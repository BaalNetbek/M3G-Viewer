M3G Viewer User Manual

This document provides information for using the M3G Viewer. Figure 1 shows what the user interface should look like upon executing the browser, M3gViewer.exe.

Figure 1: M3G Viewer

Figure 1 shows four main views: the File View, the Gaph View, the Properties view, and the Console. The File View is used to inspect the contents of the .m3g file as it is structured in the file. The Graph View is used to inspect the structure of the scene graph and the relationship between the M3G Objects. The Properties view is used to browse the properties of each M3G Object and the Console is used to display messages resulting from using the M3G Viewer.

To view a .m3g file, simply open a file on your system using the File->Open menu item (see Figure 2).

Figure 2: File Menu

The Open action will bring up a File Browser dialog panel, enabling you to select a .m3g file from your platform's file system. Once you open a file, the .m3g file will be analysed and the contents will be displayed in both the File and Graph Views. You can switch between views by selecting the tab item at the top of each view. Figure 3 shows content being displayed in the Graph View, while Figure 4 shows content being displayed in the File View.

Figure 3: Graph View with Properties displaying Texture2D selection

Figure 4: File View with Properties displaying Camera selection

Note that Figure 3 and Figure 4 also display Properties from the selected node in each view. The Properties view will be dynamically updated as the user selects M3G Objects in either the File or Graph View. The Properties view displays the attributes of each node in the selected object's hierarchy. For example, the Texture2D object in Figure 3 is derived from the Transformable object, which in turn is derived from the Object3D object. The Properties view is displaying the attributes of each of these objects since Texture2D inherits the attributes of its parents, the Transformable and Object3D objects. To understand the context of these attributes, please refer to the M3G Specification.

In Figure 2, the File->Dump menu item is used to export the contents of the .m3g file to an ASCII text file. This feature is useful for getting detailed information about each M3G Object without attempting to extract it from the Properties view. For instance, the contents of the VertexBuffer contains large arrays of values that are easier to read from the ASCII file than inspecting them via the Properties view.

A useful feature of the M3G Viewer is to extract individual objects from the scene graph into thier own file. For example lets assume that the user wants to create a valid .m3g file containing only the data representing a Mesh node and its dependencies (i.e. Appearance, VertexArray, VertexBuffer, etc.). This can be done by right-clicking on the object and selecting the Export item in the pop-up context menu (see Figure 5). The Export action will bring up a Export Wizard providing the user with an interface for creating the new .m3g file (see Figure 6).

Figure 5: Context Menu for exporting individual M3G Objects

Figure 6: The M3G Object Export Wizard

Version 0.3 of the M3G Viewer introduces a 3rd view for visualizing the .m3g file scene graph. After the .m3g file is loaded, the user may render the scene graph by selecting the Display menu item in the File or Graph view's pull-down menu (see Figure 7). The current scene graph will be rendered, presenting the data in a 3D view (see Figure 8). If the M3G Viewer is occluded by another window, sometimes the 3D Display will not be refreshed when the window is brought to the top of the window stack. In this case, use the Refresh menu item in the File or Graph view's pull-down menu to update the display.

Figure 7: 3D View Display Context Menu

Figure 8: 3D View

Finally, to exit the M3G Viewer when you are done browsing the content of your .m3g file, simply choose File->Exit from the drop-down menu (see Figure 2).
